        # temporal — Обработка ошибок

        Homework-шаблон для урока **l2_error_handling** (Обработка ошибок) на платформе Vibe Learn.

        ## Что делать

        Реализуй OnboardingWorkflow на Go SDK (go.temporal.io/sdk) с активностью CheckKYC,
которая в зависимости от входа возвращает: транзиентную ошибку (503), бизнес-ошибку
NewNonRetryableApplicationError типа "DocumentsInvalid", либо успех. В workflow на
.Get() разбери ошибку через errors.As: транзиентные дожимаются ретраями, при
"DocumentsInvalid" — ветка отклонения заявки (не выдавать карту), и обработай
CanceledError для cleanup. Тесты через TestWorkflowEnvironment с mock-активностями
проверят, что каждая ветка отрабатывает корректно и карта НЕ выдаётся при провале KYC.

## Контекст (из transfer-задачи урока)

Workflow онбординга нового клиента: создать аккаунт → проверить документы (KYC) →
выдать карту. Активность CheckKYC дёргает внешний сервис проверки. Бывает:
(1) сервис KYC недоступен (timeout/503); (2) сервис вернул "documents_invalid"
(документы реально не прошли проверку); (3) клиент сам отменил заявку (cancel).
Сейчас разработчик пишет: `err := ExecuteActivity(...).Get(...); if err != nil { return nil }`
— то есть проглатывает любую ошибку и продолжает выдавать карту.

**Вопрос:** что не так и как сделать правильно? Опиши:
(a) почему `return nil` при ошибке KYC — катастрофа;
(b) как обработать "documents_invalid" (бизнес) vs timeout/503 (транзиентное);
(c) что делать при отмене заявки клиентом (CanceledError).

## Recap из урока

- Ошибка активности всплывает в workflow на `.Get()` **только после исчерпания ретраев** (или сразу, если non-retryable). Промежуточные попытки workflow не видит.
- **Дефолт — ретраить ВСЁ.** Бизнес-ошибку («карта отклонена») надо явно пометить через `NewNonRetryableApplicationError`, иначе она зациклится в ретраях.
- Различай: **транзиентная инфра-ошибка** (timeout/503) → пусть ретраит RetryPolicy; **постоянная бизнес-ошибка** → не ретраить, ветвиться или компенсировать.
- `CanceledError` — кооперативная отмена, **лови** для cleanup. `TerminatedError` — принудительное убийство на сервере, **поймать нельзя**, cleanup не выполнится.
- Ронять workflow (`return err`) — глобально и финально (по дефолту целиком не ретраится). Компенсацию выполняй ДО возврата ошибки. Молча глотать ошибку — врать о состоянии процесса.

        ## Как работать

        1. Платформа Vibe Learn создаёт копию этого репо в твоём GitHub-аккаунте по клику «Начать домашку» на странице урока (через GitHub `/generate`, codecrafters-pattern).
        2. Склонируй копию локально, реализуй TODO в `main.go` (workflow + активности), прогони тесты, запушь.
        3. CI (`.github/workflows/ci.yml`) запускает `go vet` + `go test ./...` на каждый push. Платформа слушает результат через webhook от GitHub Actions и обновляет статус домашки на странице урока.

        ## Локальное окружение

        - Go 1.22+
        - SDK: `go.temporal.io/sdk`
        - Docker + docker-compose — `docker compose up` поднимает Temporal dev server на `:7233` + Web UI на `:8233`. Адрес переопределяется через env `TEMPORAL_ADDRESS` (дефолт `localhost:7233`).
        - Юнит-тесты на `testsuite.TestWorkflowEnvironment` (активности замоканы) бегут в CI БЕЗ сервера; интеграционный тест включается через `TEMPORAL_INTEGRATION=1`.

        ## Запуск

        ```bash
        # Поднять локальный Temporal dev server + UI
        docker compose up -d
        # Web UI: http://localhost:8233

        # Прогнать тесты (юнит на TestWorkflowEnvironment — без сервера;
        # интеграционный включается через TEMPORAL_INTEGRATION=1)
        go test ./...
        TEMPORAL_INTEGRATION=1 go test ./...

        # Запустить воркер (регистрирует workflow + активности, слушает task queue)
        go run .
        ```
